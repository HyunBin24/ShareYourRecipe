### 1. 제품 소개

사용자들이 서로 레시피를 공유할 수 있는 반응형 앱입니다.

로그인한 사용자는 레시피를 등록, 수정, 삭제할 수 있으며 마음에 드는 레시피를 즐겨찾기에 추가할 수 있습니다.

로그인하지 않은 경우에도 다른 사람들이 올린 레시피를 열람하거나 랜덤으로 레시피를 뽑아볼 수 있습니다.

앱 하단의 ‘뭐 먹지?’ 버튼을 누르면 등록된 레시피 중 하나가 무작위로 추천됩니다. 마음에 드는 레시피가 나올 때까지 계속 다시 뽑을 수 있습니다.

---

### 2. 사용 방법

1. 이메일을 이용하여 회원가입을 한다.
2. 가입한 이메일로 인증 메일을 확인을 한 후 로그인하여 사용한다.
3. 비로그인 시 레시피 열람과 레시피 뽑기 기능(하단의 '뭐 먹지?)만 사용할 수 있다.
4. 로그인 시 레시피 등록/수정/삭제와 즐겨찾기 기능 사용할 수 있다.

### 3. 주요 파일 설명

1. App.jsx
    1) 애플리케이션의 전체 경로와 페이지 이동을 관리하는 라우팅 담당 컴포넌트
    2) 주요 기능  
        (1) 라우팅 설정  
            react-router-dom을 사용하여 URL 경로와 해당 페이지 컴포넌트를 매핑
2. Start.jsx
    1) 메인 화면 역할을 담당하는 컴포넌트
    2) 주요 기능  
        (1) 레시피 목록 조회  
            - Supabase의 recipe 테이블에서 레시피를 가져옴  
        (2) 사용자 인증 및 세션 관리  
            - onAuthStateChange를 통해 로그인 상태를 감지함  
        (3) 즐겨찾기 시스템  
            - 로그인한 사용자가 레시피를 즐겨찾기할 수 있으며 Set 자료구조를 활용함  
        (4) 랜덤 레시피 추천  
            - '뭐 먹지?' 기능을 통해 등록된 레시피 중 하나를 랜덤으로 추출하여 Random 페이지로 이동  
        (5) 경로 보호  
            - 비로그인 사용자가 '등록', '즐겨찾기' 접근 시 로그인 알람과 함께 로그인(회원가입)을 유도함
3. Detail.jsx
    1) 선택된 레시피의 상세 정보를 보여주고 수정/삭제 등의 관리 기능을 제공하는 컴포넌트
    2) 주요 기능  
        (1) 상세 정보 출력  
            - UseLocation을 통해 이전 페이지로부터 전달받은 레시피의 상세 정보를 랜더링함  
        (2) 레시피 수정 및 삭제  
            - 현재 로그인 사용자의 ID와 레시피 작성자의 ID를 비교하여 본인이 등록한 레시피일 경우에만 수정/삭제 버튼이 보이도록 함  
            - 삭제 클릭 시 window.confirm을 통해 확인 후 Supabase에서 데이터를 삭제함  
        (3) 개별 즐겨찾기 관리  
            - 즐겨찾기 여부를 조회하고 페이지 내에서 즐겨찾기를 추가하거나 해제할 수 있음  
        (4) 랜덤 레시피 추천  
            - '뭐 먹지?' 기능을 통해 등록된 레시피 중 하나를 랜덤으로 추출하여 Random 페이지로 이동
4. Register.jsx
    1) 사용자가 레시피를 등록하거나 기존에 등록된 내용을 수정할 수 있는 컴포넌트
    2) 주요 기능  
        (1) 레시피 등록 및 수정  
            - useLocation을 통해 전달된 recipeEdit 유무에 따라 등록/수정을 효율적으로 처리함
        (2) 이미지 업로드 및 미리 보기  
            - 사용자가 이미지를 선택하면 URL.createObjectURL을 통해 미리 보기 이미지를 띄우고, 등록 버튼 클릭 시 Supabase Storage에 업로드함  
        (3) 상세 정보 입력  
            - 레시피 작성 시 필수 항목 누락 여부를 체크함  
        (4) 랜덤 레시피 추천  
            - '뭐 먹지?' 기능을 통해 등록된 레시피 중 하나를 랜덤으로 추출하여 Random 페이지로 이동
5. Bookmark.jsx
    1) 사용자가 즐겨찾기한 레시피 목록을 확인할 수 있는 컴포넌트
    2) 주요 기능
        (1) 북마크 목록 조회  
            - Supabase bookmark 테이블과 recipe 테이블을 통해 현재 로그인한 사용자가 즐겨찾기한 레시피 데이터만 보이도록 함  
        (2) 데이터 상태 관리  
            - userDataLoading 상태를 통해 데이토 로딩 중임을 사용자에게 알리고, 즐겨찾기 목록이 비어있을 경우 안내 메시지를 출력함  
        (3) 랜덤 레시피 추천  
            - '뭐 먹지?' 기능을 통해 등록된 레시피 중 하나를 랜덤으로 추출하여 Random 페이지로 이동
6. Random.jsx
    1) 사용자가 레시피 선택이 고민될 때 등록된 전체 레시피 중 하나를 무작위로 추천해 주는 컴포넌트
    2) 주요 기능  
        (1) 랜덤 레시피 추출  
            - Math.random()을 사용하여 랜덤으로 레시피를 추출함  
            - 이전 레시피와는 중복되지 않도록 함  
        (2) 로딩 상태  
            - 레시피 데이터 추출 및 로딩 시간 동안 로딩 중임을 사용자에게 알림
7. MY.jsx
    1) 로그인/회원가입과 로그인된 사용자의 개인 레시피를 관리하는 컴포넌트
    2) 주요 기능  
        (1) 사용자 인증 시스템  
            - Supabase Auth를 활용해 이메일 기반의 회원가입/로그인 기능을 구현함  
        (2) 사용자 레시피  
            - 로그인 성공 시 해당 사용자가 작성한 레시피 목록을 볼 수 있도록 함  
        (3) 랜덤 레시피 추천  
            - '뭐 먹지?' 기능을 통해 등록된 레시피 중 하나를 랜덤으로 추출하여 Random 페이지로 이동
8. Common.module.css
    1) 공통된 css와 모바일에 대응하기 위한 미디어 쿼리를 관리하는 스타일 시트

### 5. AI 활용 정리

1. 제미나이 (Gemini)
    1) 랜덤 함수 구현 (레시피 중복 방지 포함)
    2) Recipe.jsx / Detail.jsx에서 선택된 레시피 정보 전달
    3) 로딩 시 '로딩 중...' 화면 표시
    4) 페이지 이동 처리
    5) 체크박스 변경 함수, 이미지 등록 및 미리 보기 기능
    6) 사진 업로드 및 DB 저장 로직
    7) 이전 파일 이름 삭제 처리
2. 안티그래비티 (Antigravity)
    1) 로그인/회원가입 처리 및 세션 관리
    2) 로그인 감지 및 사용자 데이터 가져오기
    3) 즐겨찾기 목록 조회, 추가/삭제 기능
    4) 즐겨찾기 색상 변경 UI 처리
    5) 로그인한 사용자와 등록자가 동일할 때만 수정/삭제 가능하도록 제어
    6) 메일 발송 관련 기능
    7) 모바일 반응형 레시피 카드 UI (그리드, 이미지 비율 고정 등)
    8) 중복 클릭 방지 및 세션 유효성 체크
